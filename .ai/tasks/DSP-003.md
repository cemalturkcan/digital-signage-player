# DSP-003: Offline-First Minimum (Cache + Reconnect)

## Objective

Implement the practical offline-first minimum required by the case.

## Scope

- Cache playlist locally
- Cache media references and playable assets for offline continuity
- Offline cold start from cache
- Retry/backoff for playlist fetch and MQTT reconnect
- Safe fallback when cache is missing or partial

## Deliverables

- apps/player/src/app/stores/media/store.ts
- apps/player/src/app/stores/playlist/store.ts
- apps/player/src/app/request/requests/playlist.ts
- docs/OFFLINE_STRATEGY.md

## Dependencies

DSP-002

## Time Estimate

8 hours

## Acceptance Criteria

- [ ] Player continues playback from cache during network loss
- [ ] Offline cold start loads last valid cached playlist when available
- [ ] Reconnect refreshes playlist without crash
- [ ] Retry/backoff behavior is deterministic and documented

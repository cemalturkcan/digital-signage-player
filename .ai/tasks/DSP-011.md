# DSP-011: Tizen WGT Build Automation

**Status**: Completed
**Priority**: High
**Completed At**: 2026-02-27T08:57:06Z

## Summary

Automated the Tizen WGT packaging process for the player app, enabling one-command signed WGT builds with proper environment configuration and Turbo caching.

## Changes Implemented

### Build Script (`apps/player/package.json`)

- Build script now invokes: `bash scripts/package-wgt.sh`
- Root build remains: `turbo run build` (unchanged)

### Packaging Script (`apps/player/scripts/package-wgt.sh`)

- Requires `.env.tizen` file (exits if missing)
- Requires `TIZEN_CLI` environment variable (no hardcoded fallback)
- Builds via `vite build --mode tizen`
- Ensures Tizen profile exists / creates if missing
- Packages WGT and renames output to `digital_signage_player_${version}.wgt`
- Outputs artifact to `apps/player/` root (not dist)

### Turbo Configuration (`turbo.json`)

- Added `globalDependencies`: `**/.env.tizen`
- Added `tasks.build.outputs`: `*.wgt`

### Git Ignore

- Root `.gitignore` updated to ignore `.env.tizen` (in addition to existing `*.wgt`)

## Environment Requirements

```bash
# Required in .env.tizen
TIZEN_CLI=C:/path/to/tizen-studio/tools/ide/bin/tizen.bat
TIZEN_PROFILE=<profile_name>
TIZEN_AUTHOR_CERT_PASSWORD=<author_cert_password>

# Optional
TIZEN_AUTHOR_CERT_NAME=<author_alias>
```

## Usage

```bash
# From repo root
pnpm run build

# Or specifically for tizen
cd apps/player
pnpm run build
```

# DSP-002: Case Contracts and Command Validation

## Objective

Freeze the case-critical MQTT/command contract and implement strict payload validation.

## Scope

- Command/topic contract alignment between backend, player, and docs
- Required commands: reload_playlist, restart_player, play, pause, set_volume, screenshot
- Correlation handling via commandId -> correlationId
- Validation for malformed/unsupported command payloads
- Consistent command_result success/error envelopes

## Deliverables

- packages/contracts/src/commands.ts
- apps/backend/src/app/command-processor/service.ts
- apps/player/src/app/commands/bus.ts
- docs/MQTT_PROTOCOL.md

## Dependencies

DSP-004

## Time Estimate

6 hours

## Acceptance Criteria

- [ ] Invalid command payloads are rejected without crash
- [ ] Command names and payload schema are consistent in code and docs
- [ ] Every processed command produces command_result with matching correlationId
- [ ] Unsupported commands return structured error response

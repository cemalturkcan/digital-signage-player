# DSP-010: Backend Refactoring

## Objective

Stabilize backend architecture with consistent patterns for database, MQTT, logging, and API endpoints. Remove unnecessary complexity and align with project conventions.

## Scope

- Database: Refactor to Go-style lazy initialization (`let db: Pool | null` + `connectDb()`)
- MQTT: Shared client with explicit `connect()` pattern, browser websocket fixes
- Logging: Setup pino, relocate to `src/app/logger/logger.ts`
- Registration: Remove `deviceType`, `softwareVersion`, response `deviceId`
- Health endpoint: Simplify to `{status: 'ok'}`
- Docker Compose: Remove explicit networks and named volumes

## Deliverables

- Database module with lazy initialization pattern
- MQTT shared client with explicit connection lifecycle
- Pino logging infrastructure
- Simplified registration endpoint
- Minimal health check endpoint
- Cleaned up compose configuration

## Dependencies

DSP-009

## Acceptance Criteria

- [x] Database uses Go-style lazy init (Pool | null + connectDb())
- [x] MQTT uses shared client with explicit connect()
- [x] Browser websocket works (ws://:9001, MQTT*CLIENT*\* env vars)

- Backend now follows no-comments policy with self-documenting names
